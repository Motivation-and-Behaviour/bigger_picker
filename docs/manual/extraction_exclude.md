# Excluding Datasets

## Rationale

Sometimes when checking articles, you may find that a dataset should have been excluded at the review stage.
In other words, the dataset did not meet the [inclusion criteria](screening_overview.md/#inclusion-and-exclusion-criteria) and should be excluded.
The steps below outline how to make sure the dataset is correctly excluded.

## Overview of Steps

Once you have identified that a dataset should have been excluded, you will need to remove it from Airtable and Asana, and mark it as excluded in Rayyan.
The full process is below, but the general steps are:

1. Delete the dataset from Airtable
2. Delete the dataset task from Asana
3. In Rayyan, mark the dataset as excluded with the appropriate reason

???+ note
    If a dataset has more than one article attached to it, make sure that **all articles** do not meet the inclusion criteria before excluding.
    For example, if an article has a sample age over 18 but another article from the same dataset has a sample age under 18, then the dataset should **not** be excluded.

### [Rayyan](https://www.rayyan.ai/) Steps

The steps for excluding a dataset in Rayyan are the same as for [excluding during screening](screening_process.md/#full-text-screening).

1. Navigate to the **Full Text Screening** tab.
2. Use the search bar to find the dataset by its title.
3. Add an exclusion reason by clicking on the dataset, then selecting the 'Exclude with Reasons' box.
4. Click the red 'Exclude' button to finalise the exclusion.
